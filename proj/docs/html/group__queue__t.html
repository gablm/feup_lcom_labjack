Detailed Description

Container which follows the FIFO (First in, First out) principle.

This implementation uses a memory-allocated vector with a static size, which will be the queue's maximum capacity. The actual current queue size is saved to the curr_size variable. To avoid moving every element in the container when pushing or removing elements, this implementation uses two variables (front_pos and end_pos) to determine what section of the vector is valid.
